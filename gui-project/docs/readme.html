Introduction

XMLSpectrum-FX is a simple graphical front-end for controlling an XSLT transform that converts XPath and related languages to a form rendered in HTML. The application is coded in Java-FX 2.2

The Saxon-HE XSLT 2.0 processor from Saxonica is discributed with this application.

Getting Started

Installation

For the latest source file and build, see the GitHub XMLSpectrum project.

From the command-line run java -version to check that you have the latest version of Java installed on your machine - this should be at least Java 1.7.0_07-b11.

Launch

Using the command shell of your OS, navigate to the install path and run XMLSpectrum-FX with the following:

java -jar XMLSpectrum-FX.jar

Enter Content

Once launched, you will see the main XMLSpectrum-FX window. This comprises an embedded browser (Webkit) with a vertical control panel to the right of it. Above the browser you will find a text-box; enter the URL or path of a source file here, alternatively, drag-and-drop a local file from your file manager to this location.

Run Transfrom

With the text-box populated with a valid URL or file-path, press the adjacent Run button. You should now see the source code rendered in the browser view in colorised form and formatted in accordance with the parameter settings in the control panel.

Using XMLSpectrum-FX

This product's main use is as a product demonstrator and graphical test-harness for the XMLSpectrum XSLT stylesheets which are really optimised for server-side document-processing. However, it may also prove useful as an ad-hoc tool for preparing XPath/XSLT code snippets ready for publishing on the web.

Asynchronous Processing

Large files or projects may take a while to process; to ensure UI responsiveness, all transforms are performed on a separate thread. The XMLSpectrum status bar below the HTML view is updated continusouly to show the progress of a transform

Blogging Editors

After running a transform, the pre element is copied to the clipboard, ready for pasting into a blogging editor. This has been tested for WordPress and Blogger, but blogger will require a wrapping div element with a style attribute value of overflow:auto to ensure code scrolls properly.

Settings

Settings for syntax-coloring and formatting are managed from the 2 vertical panels surrounding the browser view. The following settings are available:

  • Source Choose whether to use existing a path to an existing file or copy and paste text from an editor.
  • Font Switch between Source Code Pro and the default monospaced font.
  • Color Theme Select from a range of pre-defined color themes.
  • Language For XML-based code snippets or fragments where no XML namespace is available, select the language of the source code as a hint to ensure it is rendered properly. Also, if XQuery happens also happens to be well-formed XML, you will need to explicitly select XQuery as the language.
  • XSLT Project For modular XSLT projects: The processor traverses all imported files, creating hyperlinked content for public members and presents a Table of Contents for all files in the project.
  • Auto-Trim Removes leading spaces from XML to allow correction of the formatting. Leading spaces are trimmed, not removed entirely, to ensure text indentation is still maintained, but in a way that is consistent with the XML formatting.
  • Indent Set the number of characters to indent for each level of XML nesting.
  • Force NewLines Switch this on if the source XML has no existing linefeeds between elements. Linefeeds are inserted except where mixed-content is detected.
  • Indent Mixed Content Use this option when extensive mixed content impedes the indentation and hence readability of the XML. Note that selecting this option may compromise some significant whitespace in the XML.
